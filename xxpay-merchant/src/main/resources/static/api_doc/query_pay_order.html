接口名称：支付订单查询接口

请求方式：POST

请求地址：http://domain:3030/api/pay/query_order

请将参数以json格式传在请求地址后面。http://domain:3030/api/pay/query_order?params=URLEncoder.encode(JSON.toJSONString(reqMap), "utf-8")

如：http://domain:3030/api/pay/query_order?params=%7B%22mchId%22%3A%22100000%22%2C%22mchOrderNo%22%3A%22G20190401015248000000%22%2C%22sign%22%3A%22A566F9095751B2D56F7F91BC0DFC94FA%22%7D

查询参数params为以下内容的json格式内容。
接口参数 类型 是否可空 参数说明 其他
sign string(32) 否 请求参数的签名
mchOrderNo string(30) 否 商户订单编号
mchId string(20) 否 商户号
返回参数
接口参数 类型 是否可空 参数说明 其他
sign string(32) 否 请求参数的签名
retCode string(10) 否 (返回状态码).SUCCESS/FAIL此字段是通信标识，非交易标识，交易是否成功需要结合resCode和result中status来判断
resCode string(10) 否 业务结果，SUCCESS/FAIL
retMsg string(10) 否 (返回状态码)返回信息，如非空，为错误原因 签名失败 参数格式校验错误
result string(10) 否 订单详情。base64编码的json内容。
result明细
接口参数 类型 是否可空 参数说明 其他
payOrderId string(30) 否 平台支付订单编号
mchId string(20) 否 商户号
mchOrderNo string(30) 否 商户订单编号，唯一不允许重复
channelId string(30) 否 通道
amount string(20) 否 金额，单位分
currency string(30) 是 币种
status string(30) 是 支付状态(2-支付成功。3-通知商户成功)。除2和3之外都是支付失败。
clientIp string(30) 否 clientIp
device string(30) 是 device
subject string(64) 否 商品标题
param1 string(30) 是 param1
param2 string(30) 是 param2
paySuccTime string(256) 否 支付时间
